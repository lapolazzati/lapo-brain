# lapo-brain

Second brain for Lapo. Markdown files, Git, Tolaria, Claude Code.

## Why this exists

Three live commitments need a single source of truth so nothing slips:

- **Catone** — equity bet, AI tax due diligence for Italian commercialisti
- **CEG** — equity bet, bioremediation Italian market entry
- **Klaaryo** — paying consulting, keeps the cashflow alive

Plus one weekly ritual that ties it all together.

## Folder structure

```
00-weekly/              Weekly plans + logs. The keystone. One file per ISO week.
  └── briefs/           Daily morning briefings (auto-generated by Andy)
10-projects/            One folder per active project. Notes, decisions, artefacts.
  ├── catone/           AI tax due diligence for commercialisti
  ├── ceg/              Bioremediation Italian market entry
  └── klaaryo/          International expansion consulting
20-meetings/            Granola dumps and meeting notes. Date-prefixed filenames.
30-people/              One file per key contact (Gianni, Vincenzo, Peter, etc.)
40-targets/             Quarterly and monthly OKRs
50-resources/           Reference materials and reading archives
  └── readings/         Weekly reading lists + Readwise highlights by book
90-archive/             Stuff that's no longer active but worth keeping.
```

Three project folders, three commitments. If a fourth folder shows up in `10-projects/`, something has gone wrong — either an old commitment needs killing or a new one needs a real conversation.

## The weekly rhythm (automated by Andy)

### Sunday 5pm: Weekly Planning
Andy reminds you to plan the next week:
1. Create `00-weekly/YYYY-W{N}.md` from `_template.md`
2. Fill in **Top 3 priorities** — one per project, maximum
3. **Calendar reality check** — what's scheduled vs what should be
4. Commit and push

### Monday 8am: Morning Briefing
Andy sends two things:
- **WhatsApp summary**: Quick read on phone (BUILD/SELL mode, top priorities, task counts)
- **Detailed brief**: Committed to `00-weekly/briefs/YYYY-MM-DD.md` with full Linear tasks, calendar, OKR links

### Friday 3pm/5pm/6pm: End-of-Week Review
Andy nudges you to close the week:
- What shipped
- What slipped (and why)
- What surprised you
- Carry into next week

### Friday 6pm: Weekly Reading List
Auto-generated from:
- Readwise highlights
- WhatsApp Readings group
- Newsletters from Gmail

Saved to `50-resources/readings/YYYY-MM-DD.md`

The weekly file is where Lapo and Andy collaborate. Anything that doesn't earn a slot in a weekly priority gets honest about whether it matters.

## Working with Andy (AI Project Manager)

Andy is an AI project manager built on Claude Code + nanoclaw, with full access to this repo. See `ANDY.md` for complete documentation.

### What Andy does automatically:

**Daily:**
- Auto-updates daily logs in weekly files each evening
- Tracks Linear tasks and flags blockers
- Commits meeting notes to `20-meetings/` (via Granola MCP)
- Updates `30-people/` after meaningful interactions

**Weekly:**
- Monday 8am: Morning briefing (WhatsApp + GitHub)
- Friday 3/5/6pm: End-of-week review reminders
- Friday 6pm: Reading list generation
- Sunday 5pm: Weekly planning reminder

**On request (`/sync` command):**
1. Review recent conversations and context
2. Propose file updates
3. Show diff for approval
4. Commit and push to GitHub

### Integrations:

- ✅ **Linear API**: Task tracking, OKR linking
- ✅ **Google Calendar**: Event tracking
- ✅ **Gmail**: Newsletter filtering
- ✅ **Readwise**: Highlights organized by book
- ✅ **GitHub**: Auto-commit and push
- 🔄 **Granola MCP**: Meeting transcripts (pending OAuth)
- 🔄 **Metabase**: Catone analytics (pending API key)

### Andy's guardrails:

- Never invents new top-level folders without asking
- Never archives a project without explicit say-so
- Shows diffs before committing (for non-routine commits)
- Flags when you're working on things not in OKRs
- Protects your stated priorities from daily impulses

## File naming conventions

- Weekly: `2026-W18.md` (ISO week)
- Meetings: `2026-04-29-vincenzo-tax-coach.md` (date + slug)
- People: `gianni-rizzi.md` (kebab-case)
- Projects: kebab-case folders, README.md inside each

## What this is not

- Not a place to over-build before selling.
- Not a substitute for shipping.

The system is not the work.
