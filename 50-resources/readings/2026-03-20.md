### **When Your Vibe Coded App Goes Viral—And Then Goes Down** (Every)
*Mar 20, 2026*

Dan Shipper's deep dive on building Proof (their AI document editor) entirely through vibe coding with OpenAI's Codex and GPT-5.4. Built 140K lines of code with 1,600 commits in ~2.5 months while running Every. Key insights: vibe coding is a CEO superpower, but debugging is harder because models prefer local fixes over root cause analysis. Models lack knowledge of cutting-edge best practices (Yjs/Hocus Pocus patterns weren't in training data). The feedback loop (code → test → deploy → monitor) is the bottleneck. "If you can vibe code it, you can vibe fix it. You just might not be able to fix it quickly." Codex's new subagents feature helped massively during the outage - one agent pushing fixes, another watching errors, another coding solutions.

**Key stat:** Median full-time developer made 417 commits per *year* (2020-2024). Dan made 1,600 in 2.5 months part-time.
