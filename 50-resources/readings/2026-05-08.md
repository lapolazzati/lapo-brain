### **Claude Code for Product Managers** (Every - Marcus Moretti)
*May 1, 2026*

**How general manager runs Spiral using AI for planning, monitoring, and research:**

Marcus Moretti, GM of Spiral (Every's AI writing partner), describes how Claude Code eliminated PM drudgery. The only product document he writes is the roadmap—everything else (PRDs, tickets) is written by Claude.

**Key workflows:**
- **/prioritize command** - Claude sweeps GitHub daily, checks stale tickets, confirms "now" column is this week's work, pulls urgent items from backlog
- **/pulse command** - Delivers analyst briefing instead of dashboard charts. Surfaces metrics (active users, chats/messages/drafts, response times, conversation quality 1-5, anonymized use cases). Claude reads text, grades conversations, flags anomalies, explains findings in plain English
- **Roadmap to 100 tickets in minutes** - From detailed roadmap, Claude used GitHub CLI to generate ~100 detailed tickets with strategic context, supporting data, acceptance criteria, technical notes. Unbiased prioritization.

**Research workflow:** Used Arxiv MCP to find recent stylometry papers, informed Spiral's style transfer system. Academic citations now appear in product requirements.

**What SaaS survives:** Products with (1) unique sources of critical data and (2) seamless agent integrations. GitHub, Stripe, PostHog, Logfire have MCPs and play nicely with Claude. Services without MCPs got cancelled.

**Result:** Spiral's API chats now outnumber web chats (milestone hit 3 days after launching API agent). Users consuming Spiral across their workflows, not just in the Spiral app.
